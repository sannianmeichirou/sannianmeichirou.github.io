    function factorial( num ){
        if ( num == 1 )
        {
            return 1;
        }
        return num * factorial(num-1);
    }            
   1:f（4）.return 4*f(3)   2:f(3).return 3*f(2)    3:f(2).return 2*f(1)    4:f(1).return 1

闭包：(满足下面两点)

1.函数嵌套函数
2.内部函数使用了外部函数的参数或者变量

闭包的特性：

被使用的参数和变量不会被垃圾回收机制回收，会永久的驻扎内存中

模块化代码

函数执行一次之后 用不到的变量和参数 会被js作为垃圾回收 内存中不再存在
    例：    a();
        function a(){
            var y = 5;
            y = 2 + y;
            alert( y );
        };                        y在执行之后就被垃圾回收了  不会存在内存当中

js中的正则表达式

一个匹配规则 （用来检测字符串满不满足这个规则的东西）

规则定义 ###：

    var reg = /qwe/;                 /匹配规则/                  
    var reg = new RegExp('qwe');     new RegExp（'匹配规则'）  
reg.test()   检测是否满足规则，如果满足规则，返回true，否则返回false
str.match()  满足匹配规则，把匹配成功的内容放到一个数组里面返回数组，否则返回null    
例：
       var reg = /qwe/;       匹配‘qwe’字符串
       var str = 'qweaaaaa';
       alert ( reg.test(str) );    //true        
       alert ( str.match(reg) );  // [qwe]

转义 \

    \s 单个空格            \S  非空格          均指的单个
    \d 单个数字            \D  非数字        
    \w  字符（字母 数字 _）     \W  非字符
    \b  独立部分（空格 开始 结束 -）\B  非独立部分 
    \跟上特殊意义的字符  正则里面的特殊字符 \ / { } + . ? * ( ) [ ] ^ $   \加上特殊符号  相当于匹配这些特殊符号
    \r 换行        \n 回车      \t 制表符相当于按一次tab
例：
       var reg = /\d/;  匹配单个数字        var reg = /\//;     匹配特殊符号/
       var str = 'qb2ca1';                var str = 'qb2/ca1';
       alert( str.match( reg ) );  //[2]        alert( str.match( reg ) );  //[/]

量词：{}

        {1,5}     :  1 <= n <= 5     1个到5个之间
    +    表示{1,} :  n >= 1    一个及以上    
    *    表示{0,} :  n >= 0    零个及以上
    ?    表示{0,1}:  0 <= n <= 1 零个或者一个
例：
       var reg = /w+/;  匹配一个以上的w
       var str = 'qqqwwwadqqw';    
       alert( str.match( reg ) );  //[www]


标识：    /条件/标识  标识写在结尾/后面 
    g   :   全局匹配 （从头到尾匹配完）  没有g则一次匹配成功后结束 有g则全局匹配成功后结束
    i   :   不区分大小写
例：    
    var reg = /w+/gi;    全局匹配一个及以上的w 且不区分大小写 
    var str = 'qqqwwWadqqw';
    alert( str.match( reg ) );  //[wwW,w]

子集： ()

    ()内为子集内容 匹配时子集作为整体匹配    注意！当有子项，并且没有全局标识的时候，match返回的数组的第二项代表子集内容
例：
    var reg = /(qw)+(ss)/;        //匹配子项1 (qw) 匹配子项2 (ss)        
    var str = 'qwqwssqqwsqwqwqwss';
    alert( str.match( reg ) );     //[qwqw,ss,qw,ss] 后面的qw,ss 为返还的子集内容

字符集： [ ]

    [0-9]  ： 从0到9    [1-5]  :  从1到5        
    [a-z]  :  a - z        [f-k]  :  f - k
    [A-Z]  :  A - Z
    []里内容为其他字符时候 表示或者
    字符集里面的 除了转义\  其他的 特殊字符 不特殊  是单纯的没有意义的字符            
例：
    var reg = /[(+q/{\de1,5}w)]/g;   全局匹配 （或者+或者q或者/或者{ 或者 单个数字 或者e或者1或者，或者5....
    var str = '(q+d{wdeq/w3fds\\';
    alert( str.match( reg ) );     // [(,q,+,{,w,e,q,/,w,3]

web笔记之js8 绑定捕获

发表于 2017-04-15 | 分类于 web笔记js

事件捕获

事件捕获： 专门对于 addEventListener()这个绑定事件的方法而言
       即 ie8+ 和其他浏览器才有 事件捕获的说法
       例： box1>box2>box3                         // f1 f2 f3 表示分别弹出 1 2 3 
        oBox2.addEventListener('click' , f2 , false);
        oBox3.addEventListener('click' , f3 , false);
        oBox1.addEventListener('click' , f1 , false);
    false 表示不捕获   结果为  3  2  1  冒泡 由里到外 与顺序无关
    true  表示捕获          
        oBox2.addEventListener('click' , f2 , true);        oBox2.addEventListener('click' , f2 , false);
        oBox3.addEventListener('click' , f3 , true);        oBox3.addEventListener('click' , f33 , false);
        oBox1.addEventListener('click' , f1 , true);        oBox3.addEventListener('click' , f3 , true);
        oBox2.addEventListener('click' , f22 , false);        oBox1.addEventListener('click' , f1 , true);    
        oBox3.addEventListener('click' , f33 , false);        oBox2.addEventListener('click' , f22 , false);
        oBox1.addEventListener('click' , f11 , false);      oBox1.addEventListener('click' , f11 , false);
    假设父级有捕获的事件，            //1  2  3  33 22 11            // 1 33 3 2 22 11
    先执行最高级父级的捕获事件，
    再从低到高执行父级的不捕获事件。    ！！即由外到里（捕获） 再由里到外（不捕获 冒泡）
    事件源（同时有捕获和不捕获）：看捕获和不捕获的代码位置顺序，谁在前面谁先

事件解绑

绑定事件：attachEvent  ie兼容 chrome 不兼容
            addEventListener  chrome兼容
      事件解绑： 即让绑定的事件不再发生
           obj.detachEvent（'on'+eName,fn） ie兼容  注意  这里的fn 要与obj.attachEvent（'on'+eName,fn）    
            中的fn 完全对应相同 即内存地址也相同  所以在这里的两个fn 不能直接写function(){}
           obj.removeEventListener(eName,fn,false) chrome兼容

事件冒泡

事件冒泡： 从子元素上产生事件的时候，事件会传递到父元素上  冒泡事件由里到外
       例：点击事件：
            oChild.onclick = fn;            
            oWrap.onclick = fn;
            function fn(){
                alert( this.id );
            }  
        child是wrap的子元素 在wrap内  结果: 点击一次child  弹框两次 第一次出现 child 第二次出现 wrap

阻止冒泡：
       e.cancelBubble  默认false 也就是不阻止冒泡     e指事件event    
       e.stopPropagation() 阻止冒泡 没有参数 
兼容写法： e = e || window.event;
       e.stopPropagation?e.stopPropagation():e.cancelBubble=true; 
       兼用情况有待重新测试  目前cancelBubble ie兼容 但是会出现新问题
                 stopPropagation() ie不兼容
